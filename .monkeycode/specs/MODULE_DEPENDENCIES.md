# 跨模块依赖关系说明

**更新日期**: 2026-05-10  
**版本**: v1.0

## 第一部分：模块依赖总览

### 1.1 依赖关系图

```
销售模块 ────┬───→ 商城模块 (客户数据同步)
             ├───→ 云仓模块 (库存查询)
             └───→ 财务模块 (业绩统计)

商城模块 ────┬───→ 支付模块 (支付回调)
             ├───→ 客服模块 (订单状态)
             ├───→ 云仓模块 (库存扣减)
             └───→ 直配模块 (配送信息)

云仓模块 ────┬───→ 采购模块 (库存预警)
             ├───→ 财务模块 (成本核算)
             └───→ 数据中心 (库存报表)

采购模块 ────┬───→ 供应商模块 (订单同步)
             ├───→ 云仓模块 (入库通知)
             └───→ 财务模块 (应付账款)

财务模块 ←───┼─── 所有业务模块 (资金流水)
             └───→ 数据中心 (财务报表)

人事模块 ────┬───→ 销售模块 (人员信息)
             ├───→ 云仓模块 (仓管员信息)
             └───→ 直配模块 (配送员信息)

数据中心 ←───┴─── 所有业务模块 (数据聚合)
```

## 第二部分：详细依赖关系

### 2.1 销售模块依赖

#### 2.1.1 依赖其他模块
- **人事模块**: 获取销售人员、经理信息
- **云仓模块**: 查询商品库存信息
- **商城模块**: 同步客户基础信息

#### 2.1.2 被其他模块依赖
- **财务模块**: 获取销售业绩数据用于提成计算
- **数据中心**: 获取拜访记录、客户开发数据
- **人事模块**: 提供销售人员绩效数据

#### 2.1.3 关键接口
```typescript
// 从人事模块获取销售人员信息
GET /api/hr/staff/sales-team

// 从云仓模块查询商品库存
GET /api/warehouse/inventory?sku={sku}

// 向财务模块推送销售业绩
POST /api/finance/sales-performance
```

### 2.2 商城模块依赖

#### 2.2.1 依赖其他模块
- **支付模块**: 发起支付、处理支付回调
- **云仓模块**: 查询商品库存、扣减库存
- **客服模块**: 创建订单售后申请
- **直配模块**: 获取配送线路和时间

#### 2.2.2 被其他模块依赖
- **销售模块**: 客户下单行为数据
- **数据中心**: 订单、销量统计数据
- **财务模块**: 订单金额、退款数据

#### 2.2.3 关键接口
```typescript
// 向支付模块发起支付
POST /api/payment/create-order

// 从云仓模块查询库存
GET /api/warehouse/inventory/available?sku={sku}

// 向云仓模块扣减库存
POST /api/warehouse/inventory/deduct

// 向直配模块创建配送任务
POST /api/delivery/create-task
```

### 2.3 云仓模块依赖

#### 2.3.1 依赖其他模块
- **采购模块**: 获取采购订单信息
- **商城模块**: 获取销售订单信息
- **人事模块**: 获取仓管员排班信息

#### 2.3.2 被其他模块依赖
- **销售模块**: 库存查询
- **商城模块**: 库存扣减
- **采购模块**: 库存预警触发采购
- **财务模块**: 商品成本核算
- **数据中心**: 库存报表数据

#### 2.3.3 关键接口
```typescript

// 向财务模块提供成本数据

```

### 2.4 采购模块依赖

#### 2.4.1 依赖其他模块
- **云仓模块**: 获取库存数据、入库确认

## 第三部分：数据同步机制

### 3.1 实时同步
- **用户认证**: JWT Token + Redis缓存
- **库存扣减**: 分布式锁 + 事务保证
- **支付回调**: Webhook + 幂等性处理

### 3.2 异步同步
- **业绩统计**: 消息队列 + 批量处理
- **报表生成**: 定时任务 + 缓存预热
- **日志收集**: 日志队列 + 异步写入

### 3.3 缓存策略
- **热点数据**: Redis缓存，TTL 5分钟
- **配置数据**: Redis缓存，TTL 1小时
- **报表数据**: Redis缓存，TTL 24小时

## 第四部分：错误处理和容错

### 4.1 网络异常处理
- **重试机制**: 指数退避重试（最多3次）
- **降级策略**: 本地缓存 + 默认值
- **熔断机制**: Hystrix模式，失败率>50%熔断

### 4.2 数据一致性保证
- **分布式事务**: TCC模式（Try-Confirm-Cancel）
- **最终一致性**: 消息队列 + 补偿事务
- **幂等性设计**: 请求ID去重

### 4.3 监控告警
- **接口成功率**: <99.9% 触发告警
- **响应时间**: >2s 触发告警
- **依赖服务**: 健康检查 + 自动切换
